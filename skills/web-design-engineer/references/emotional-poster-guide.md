# Emotional Poster Guide — 情绪海报生成系统

当任务涉及**观点输出、口播稿、短视频视觉素材、社媒传播内容**时，使用这个系统替代常规设计流程。

> 这个系统不是替代原有的 29 个配方，而是在原有体系上叠加一个"情绪控制层"。选择 Emotional Poster 学校后，先确定三个 scale，再选具体配方。

---

## 三个 Scale

### Emotion Scale：情绪强度

控制文案和视觉的情绪浓度。

| Level | 名称 | 说明 | 示例 |
|---|---|---|---|
| 1 | 冷静解释 | 客观陈述，无情绪色彩 | "AI 正在改变创作方式。" |
| 2 | 轻微态度 | 有倾向但不激烈 | "AI 确实在改变创作方式，但不是你想的那样。" |
| 3 | 明确立场 | 有观点、有判断 | "大多数创作者还没意识到，AI 改变的不是工具，是标准。" |
| 4 | 强对抗 | 制造冲突、挑战常识 | "真正被 AI 淘汰的不是创作者，而是那些只会'搬运感觉'的人。" |
| 5 | 爆发式 | 压迫感、争议感、让人想转发或反驳 | "你以为你在创作？你只是在用更贵的工具，生产更平庸的内容。" |

### Conflict Scale：对抗强度

控制内容中的"冲突结构"。

| Level | 名称 | 说明 | 结构 |
|---|---|---|---|
| 1 | 信息说明 | 平铺直叙 | A 是 B |
| 2 | 问题指出 | 引发思考 | A 应该是 B，但实际是 C |
| 3 | 反常识 | 打破预期 | 你以为 A 是 B？其实 A 是 C |
| 4 | 正反对撞 | 制造对立 | A 阵营说 B，但 C 阵营证明了 D |
| 5 | 群体/时代/利益冲突 | 升维对抗 | 整个行业都在做 A，但真相是 B，因为 C |

### Visual Scale：视觉冲击力

控制 HTML 的审美强度。

| Level | 名称 | 说明 | 视觉语言 |
|---|---|---|---|
| 1 | 普通排版 | 干净、可读、无特殊处理 | 标准卡片、正文列表 |
| 2 | 卡片排版 | 有层次感、有间距系统 | 阴影卡片、分栏布局 |
| 3 | 杂志感 | 强排版、有节奏、留白有意图 | 大标题、衬线字体、结构化留白 |
| 4 | 海报感 | 压迫、冲击、记忆点 | 超大文字、暗色底、光晕、高对比 |
| 5 | 电影预告片感 | 情绪海报、每一屏都是一个镜头 | 多层叠加、噪点、错位、几何、极端字号比 |

## Scene 架构

每个 Emotional Poster 输出由多个 **scene** 组成。每个 scene 是一个独立的视觉镜头。

**强制结构**：
```
Scene 1: 钩子 (Hook)
  - 一句话，最大字号，情绪最浓
  - 必须让人想继续看

Scene 2: 旧认知 (Old Belief)
  - 你以为的真相
  - 用 kicker 标签标注

Scene 3: 新现实 (New Reality)
  - 真正的真相
  - 用 hit 高亮关键反转

Scene 4: 冲突高潮 (Conflict Peak)
  - 正反对撞
  - 最大视觉冲击

Scene 5: 结论 (Conclusion)
  - 你的观点
  - 简洁、有力

Scene 6: 记忆点 (Memory Hook)
  - 结尾留一个让人忘不掉的东西
  - 可以是问题、预言、讽刺
```

**Scene CSS 强制规范**：
```css
.scene {
  min-height: 100vh;
  padding: 48px 24px;
  display: flex;
  flex-direction: column;
  justify-content: center;
  position: relative;
  overflow: hidden;
}
```

## 移动端优先

**强制**：
- 默认视口 390px–430px
- 所有字号使用 `clamp()` 流式缩放
- 禁止横向滚动
- touch target ≥ 44px
- 每个 scene 是一个完整屏幕

---

## 内容结构规则

### 开场 (Hook)
- **禁止**解释背景、介绍主题、铺垫
- **必须**直接进入冲突
- 示例："你以为的XX，都是错的。" / "这个行业，正在被一个人改变。"

### 中段 (Conflict)
- **必须**制造"旧认知 vs 新现实"的对抗
- 每个 scene 只传达一个信息
- 关键词必须用 hit 高亮

### 结尾 (Memory Hook)
- **禁止**总结全文、重复观点
- **必须**留下一个强记忆点
- 可以是：反问、预言、讽刺、数据冲击、未完成的故事

---

## 文案原则

- 短句优先：每句 ≤ 20 字
- 口语化：像在说话，不是在写文章
- 有钩子：每句话都要让人想看下一句
- 适合口播：文字节奏要能对上语音节奏
- 禁止：专业术语堆砌、长段落、书面语

---

## 审美禁区 (所有 Emotional Poster 配方通用)

- ❌ 普通博客排版
- ❌ 温和卡片 (rounded + shadow)
- ❌ 默认 Tailwind 布局
- ❌ 蓝白 SaaS 风
- ❌ 信息图模板感
- ❌ 像 PPT / 简历 / 公众号文章
- ❌ emoji 作为图标替代
- ❌ 外部图片依赖
- ❌ 平铺直叙的信息罗列
